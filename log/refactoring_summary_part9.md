# Сводка рефакторинга (Часть 9): Архитектурное планирование

### Цель
Спроектировать архитектуру для поддержки мульти-арендности, системы ролей (RBAC), подписок и real-time уведомлений, оставаясь в рамках стека Dart/Flutter.

### Что сделано
- Проведен детальный анализ требований к новой функциональности.
- Сравнены подходы к хранению изображений, выбран метод с использованием внешнего Object Storage как наиболее масштабируемый.
- Проанализирована целесообразность использования Dart Frog в сравнении с FastAPI. Принято стратегическое решение продолжать разработку на Dart Frog для сохранения преимуществ единой экосистемы.
- Спланирован поэтапный переход на новую архитектуру, включая изменения в БД и реализацию WebSocket.

### Результат
Сформирован четкий, поэтапный план для дальнейшей разработки, который ляжет в основу последующих изменений в коде и базе данных.
